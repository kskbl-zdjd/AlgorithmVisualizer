1. 项目简介

AlgorithmVisualizer 是一个基于 C++/Qt6 的算法可视化教学桌面应用，支持六大排序算法的逐步动画演示，并提供开发者模式——用户可自由编写任意 C++ 代码，由 Clang AST 自动分析、插桩并可视化执行过程。

入门模式

下拉选择算法 → 输入数据 → 播放演示。适合算法初学者，无需编写代码。

开发者模式

自由编写 C++ 代码 → Clang AST 自动分析 → 编译为 DLL 执行 → 多数组可视化回放。

3. 第一步：安装 Qt 6.8.3

3.1 下载 Qt Online Installer

打开浏览器，访问 Qt 官方下载页面：

https://www.qt.io/download-qt-installer

选择 Windows 平台的在线安装程序，下载并运行。

3.2 安装步骤

务必勾选 MinGW 64-bit！如果只选了 MSVC 版本，将无法使用本项目的编译脚本。 展开 Qt 6.8.3 后，确保 MinGW 64-bit 处于勾选状态。 同时也勾选 CMake 和 Ninja（在 Developer and Designer Tools 分类下）。

3.3 验证安装

安装完成后，确认以下目录存在：

C:\Qt\6.8.3\mingw_64\        ← Qt 库和 DLL
C:\Qt\Tools\mingw1310_64\    ← MinGW 编译器
C:\Qt\Tools\CMake_64\        ← CMake
C:\Qt\Tools\Ninja\           ← Ninja
如果你的 Qt 安装路径不是 C:\Qt，记下实际路径。后面的 CMakeLists.txt 配置中需要用到。

4. 第二步：安装 Clang（LLVM）

Clang 仅在开发者模式中使用，用于对用户编写的 C++ 代码进行 AST 语法分析。入门模式不需要 Clang。

4.1 推荐方案：LLVM 官方安装包

  1. 访问 LLVM 发布页面：https://github.com/llvm/llvm-project/releases
  2. 找到最新稳定版本（如 LLVM 18.x 或 19.x）
  3. 下载 LLVM-*-win64.exe 安装包
  4. 运行时选择 "Add LLVM to the system PATH"（重要！）

4.2 验证 Clang 安装

打开 PowerShell 或 命令提示符，输入：

clang --version

应输出版本信息，例如：

clang version 18.1.8
Target: x86_64-pc-windows-msvc
Thread model: posix
如果提示 "clang 不是内部或外部命令"，说明 PATH 未正确设置。请跳到常见问题排查。

5. 第三步：配置 CMakeLists.txt 路径

打开项目根目录下的 CMakeLists.txt，找到并修改以下两行，将路径改为你的 Qt 安装目录：

# === 原始内容（需要修改）===
set(Qt6_DIR "E:/QT5/qt5/lib/cmake/Qt6")
set(CMAKE_PREFIX_PATH "E:/QT5/qt5;${CMAKE_PREFIX_PATH}")

# === 修改为你的实际路径，例如：===
set(Qt6_DIR "C:/Qt/6.8.3/mingw_64/lib/cmake/Qt6")
set(CMAKE_PREFIX_PATH "C:/Qt/6.8.3/mingw_64;${CMAKE_PREFIX_PATH}")
路径对照表：
如果 Qt 装在 C:\Qt，则路径为 C:/Qt/6.8.3/mingw_64
如果 Qt 装在 D:\Qt6，则路径为 D:/Qt6/6.8.3/mingw_64
注意使用正斜杠 /，不要用反斜杠 \。

同样需要修改 build.ps1 中的 Qt 根路径（如果使用构建脚本）：

# 修改这一行
$QtRoot = "C:\Qt"   // 改为你的 Qt 安装根目录

6. 第四步：构建项目

有两种方式构建项目：

方式 A：使用构建脚本（推荐）

在项目根目录打开 PowerShell，运行：

.\build.ps1

脚本会自动完成清理 → CMake 配置 → Ninja 编译 → 复制运行时 DLL。

方式 B：手动构建

# 1. 设置环境变量
$env:PATH = "C:\Qt\6.8.3\mingw_64\bin;C:\Qt\Tools\mingw1310_64\bin;C:\Qt\Tools\CMake_64\bin;$env:PATH"

# 2. 创建 build 目录
mkdir build; cd build

# 3. CMake 配置
cmake .. -G "Ninja"

# 4. 编译
ninja

# 5. 复制运行时 DLL
copy C:\Qt\6.8.3\mingw_64\bin\*.dll bin\
xcopy /E C:\Qt\6.8.3\mingw_64\plugins\platforms bin\platforms\
编译成功后，可执行文件位于 build\bin\AlgorithmVisualizer.exe。 首次 CMake 配置可能需要 10-30 秒，编译需要 30-120 秒（视机器性能而定）。

7. 第五步：运行与功能使用

7.1 启动应用

双击 build\bin\AlgorithmVisualizer.exe 运行程序。主窗口包含两个模式页面，通过顶部工具栏切换。

入门模式

1. 左侧下拉框选择算法（冒泡/选择/插入/快速/堆/归并）

2. 右侧输入数据或用"随机生成"按钮

3. 点击"开始"播放演示

4. 使用播放控件控制速度、暂停、单步

开发者模式

1. 左侧代码编辑器编写任意 C++ 代码

2. 点击"运行"按钮

3. 程序自动分析、编译、执行并可视化

4. 使用播放控件回放执行过程

7.2 开发者模式支持的代码示例

开发者模式支持以下 C++ 操作的自动可视化：

类别支持的操作
数组操作swap, compare, assign, read
容器操作push_back, pop_back, insert, erase, clear, resize, emplace_back, at
STL 算法sort, reverse, fill, binary_search, find, copy, remove, replace, rotate
跨数组cross compare, cross assign, container swap
函数调用引用参数别名路由（参数传递跟踪）
项目 test_codes/ 目录下有 8 个测试文件，覆盖所有操作类型。可以将其中的代码粘贴到开发者模式中体验。

8. 常见问题排查

Q1: 编译报错 "Qt6 not found"

原因：CMakeLists.txt 中的 Qt 路径与你的实际安装路径不一致。
解决：修改 CMakeLists.txt 中 Qt6_DIR 和 CMAKE_PREFIX_PATH 为你的 Qt 安装路径（参见第 5 步）。

Q2: 运行 exe 报 "缺少 Qt6Core.dll" 等 DLL 错误

原因：运行时 DLL 未复制到 exe 同目录。
解决：将 C:\Qt\6.8.3\mingw_64\bin\*.dll 复制到 build\bin\ 目录，并创建 build\bin\platforms\ 和 build\bin\styles\ 子目录放入相应插件 DLL。或直接运行 build.ps1 自动完成。

Q3: 开发者模式运行时报 "clang not found"

原因：Clang 未安装或未加入 PATH。
解决：
  • 方案 A：安装 LLVM，勾选 "Add to PATH"
  • 方案 B：如果已安装 Visual Studio 2022，可在 C:\Program Files\Microsoft Visual Studio\2022\Community\VC\Tools\Llvm\x64\bin\ 找到 clang.exe
  • 方案 C：通过 MSYS2 安装：pacman -S mingw-w64-x86_64-clang

Q4: 编译链接报 "Permission denied"

原因：AlgorithmVisualizer.exe 正在运行，编译器无法写入。
解决：关闭正在运行的程序，然后重新编译。

Q5: 修改 CMakeLists.txt 后编译不生效

原因：CMake 缓存了旧配置。
解决：删除 build/ 目录，重新运行 CMake 配置 + Ninja 编译。

9. 附录：Clang 安装备选方案

如果不想安装完整的 LLVM，以下备选方案也可满足需求：

方案说明路径示例
Visual Studio 2022 VS 自带 LLVM 工具链 C:\Program Files\Microsoft Visual Studio\2022\Community\VC\Tools\Llvm\x64\bin\clang.exe
MSYS2 包管理器安装 C:\msys64\mingw64\bin\clang.exe
LLVM 官方 独立安装包（推荐） C:\Program Files\LLVM\bin\clang.exe
PATH 中的 clang 任意方式，只要终端能执行 clang 程序会尝试系统 PATH 查找
程序的 Clang 搜索顺序：
1. VS 2022/2026 自带 LLVM（Community/Professional/Enterprise/BuildTools）
2. MSYS2（mingw64/ucrt64/clang64）
3. MinGW 安装目录
4. 系统 PATH 中的 clang 命令
只要以上任一位置有 clang.exe 即可运行开发者模式。
